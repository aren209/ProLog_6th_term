# Решение задачи коммивояжёра на Prolog

### Работа 5, Задача 6 (Файл `flights.pl`)

* **Условие:** Пусть имеются N пунктов (аэропортов), для каждого из которых известно неотрицательное расстояние до каждого из других. Требуется построить кратчайший маршрут, проходящий через каждый из N пунктов ровно один раз. Можно ограничиться случаем N ≤ 10.
* **Пример:** Запрос `answer(Path, Dist).` для графа из 7 узлов вернет кратчайший маршрут в переменной `Path` (например, `[a, f, e, g, c, d, b]`) и его длину в переменной `Dist`.

---

### Используемые встроенные предикаты (Справочник)

Для реализации алгоритма используются следующие базовые функции (предикаты) языка Prolog:

* `member(Element, List)`
    * **Описание:** Проверяет, входит ли элемент `Element` в список `List`. 
    * **Применение в коде:** Используется с оператором отрицания `\+ member(Y, Visited)`, чтобы гарантировать, что мы не посетим один и тот же аэропорт дважды.

* `length(List, N)`
    * **Описание:** Вычисляет длину списка `List` и записывает её в `N`, либо проверяет, равна ли длина списка заранее заданному числу `N`.
    * **Применение в коде:** Используется в базовом случае рекурсии, чтобы понять, что длина списка посещенных аэропортов достигла общего количества узлов в графе.

* `reverse(List1, List2)`
    * **Описание:** Принимает список `List1` и возвращает список `List2`, в котором элементы расположены в обратном порядке.
    * **Применение в коде:** Поскольку при обходе графа мы добавляем новые аэропорты в начало списка (аккумулятора), итоговый путь получается записанным задом наперед. `reverse` возвращает ему правильный хронологический порядок.

* `setof(Template, Goal, SortedList)`
    * **Описание:** Предикат, который находит все возможные решения для цели `Goal`, форматирует их по шаблону `Template`, удаляет дубликаты и **автоматически сортирует** элементы по возрастанию, помещая их в `SortedList`.
    * **Применение в коде:** Используется для поиска всех возможных путей. Благодаря шаблону `Dist-Path`, список сортируется по расстоянию (от меньшего к большему), что позволяет мгновенно извлечь самый короткий маршрут из головы списка.

---

### Архитектура решения

* **`edge/3`**: Делает граф неориентированным, позволяя перемещаться между узлами в обе стороны.
* **`airport_count/1`**: Динамически вычисляет общее количество уникальных узлов (N) в графе, избавляя от необходимости жестко задавать это значение в коде.
* **`fly/4`**: Основной рекурсивный "движок" алгоритма. Перемещается по узлам, накапливает пройденное расстояние и формирует список посещенных пунктов. Оснащен базовым случаем для успешной остановки по достижении всех пунктов.
* **`solve_path/2`**: Предикат-обертка. Выбирает стартовую точку и запускает рекурсивный поиск любого валидного маршрута.
* **`answer/2`**: Точка входа в программу. Собирает все найденные маршруты, сортирует их и возвращает единственный оптимальный вариант с минимальной дистанцией.
